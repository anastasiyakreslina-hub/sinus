# Sinus — платформа для самостоятельной подготовки к ЕГЭ по профильной математике
Sinus - веб-платформа для самостоятельной подготовки к ЕГЭ по профильной математике, объединяющая банк задач, теоретические материалы и систему отслеживания прогресса пользователя.
Проект создан как единое пространство для обучения, в котором ученик может изучать теорию, решать задания и анализировать результаты без использования разных сервисов и таблиц.

Платформа: https://sinus.onrender.com/about  
GitHub: https://github.com/anastasiyakreslina-hub/sinus.git  

## Проблема и актуальность
В процессе подготовки к ЕГЭ ученики сталкиваются со следующими трудностями:
- теория, задания и результаты хранятся в разных источниках  
- данные с прогрессом часто создаются вручную  
- онлайн школы не всегда обеспечивают индивидуальный подход  
- самостоятельная подготовка требует высокой организации и дисциплины  
Sinus решает эти проблемы, объединяя ключевые элементы подготовки в одной платформе и позволяя выстраивать индивидуальный темп обучения.

## Функционал
### Пользовательская часть
- регистрация и авторизация  
- создание и редактирование профиля  
- установка цели по баллам  
- загрузка аватара  
### Обучение
- банк задач ЕГЭ по профильной математике  
- автоматическая проверка ответов  
- раздел «Работа над ошибками»  
- теоретические материалы по темам  
- привязка теории к заданиям  
### Аналитика
- отслеживание количества решённых задач  
- процент правильных ответов  
- статистика по номерам заданий  
- анализ прогресса пользователя  
### Административная часть
- добавление задач  
- удаление задач  
- редактирование задач  
- управление теоретическими материалами  

## Архитектура проекта
### Frontend
- HTML, CSS, JS
- отображение интерфейса  
- взаимодействие с пользователем  
- отправка запросов на сервер  
### Backend
- Flask (Python)  
- обработка бизнес-логики  
- управление авторизацией и сессиями  
- проверка ответов пользователей  
- работа с базой данных  
### Database
- SQLite  
- хранение пользователей, задач, результатов и теории  

## Структура базы данных
### users
- id - идентификатор пользователя  
- username - логин  
- password - хешированный пароль  
- goal - цель по баллам  
- role - роль пользователя  
- avatar - изображение профиля  

### tasks
- id - идентификатор задачи  
- number - номер задания ЕГЭ  
- source - источник задачи  
- text - условие задачи  
- solution - решение  
- answer - правильный ответ  

### user_tasks
- id - запись результата  
- user_id - пользователь  
- task_id - задача  
- status - результат решения  

### theory_table
- block_id - идентификатор блока теории  
- title - название темы  
- task_number -номер задания  
- text - текст теории  
- pdf_path - дополнительный материал  

## Логика
1. Пользователь выбирает задачу  
2. Вводит ответ  
3. Frontend отправляет запрос на backend  
4. Backend получает правильный ответ из базы данных  
5. Выполняется сравнение ответов  
6. Результат сохраняется в user_tasks  
7. Пользователь получает обратную связь  
8. Обновляется статистика и прогресс
